就将证书中的公钥取出来，然后就是和之前一样的流程了，至于ssl pinning目的就是为了防止中间人攻击

值，保存到客户端中，到时候验证，是将当前返回的证书公钥取出来，同样作哈希，然后和之前map中存入的域名和
公钥哈希进行比对，看了下主要比对逻辑在CertificatePinner类中的check方法  
```
 public void check(String hostname, List<Certificate> peerCertificates)
      throws SSLPeerUnverifiedException {

    Set<ByteString> pins = findMatchingPins(hostname);

    if (pins == null) return;

    for (int i = 0, size = peerCertificates.size(); i < size; i++) {
      X509Certificate x509Certificate = (X509Certificate) peerCertificates.get(i);
      if (pins.contains(sha1(x509Certificate))) return; // Success!
    }

    // If we couldn't find a matching pin, format a nice exception.
    StringBuilder message = new StringBuilder()
        .append("Certificate pinning failure!")
        .append("\n  Peer certificate chain:");
    for (int i = 0, size = peerCertificates.size(); i < size; i++) {
      X509Certificate x509Certificate = (X509Certificate) peerCertificates.get(i);
      message.append("\n    ").append(pin(x509Certificate))
          .append(": ").append(x509Certificate.getSubjectDN().getName());
    }
    message.append("\n  Pinned certificates for ").append(hostname).append(":");
    for (ByteString pin : pins) {
      message.append("\n    sha1/").append(pin.base64());
    }
    throw new SSLPeerUnverifiedException(message.toString());
  }
```

，继承嘛，懂得都懂，同时这个pathList字段中，它本身有个字段是element类型叫element ，是一个数组，基本跟着系统源码走了一遍
，就是去分割每个dex，然后将每个dex都对应上一个dexfile对象，再存入element数组中，每次当我们去findclass时，会先去遍历这个
element数组，先找到同名的类，就把它加载进来，那么如果我们热修复想要修复的话，那么是不是可以将我们的修复类，放到遍历element数组
的前面，这样就会先找到我们修复的类了，后面就默认不执行了，然后阿里的直接替换，是直接替换artmethod这个结构体，就是dex中所有的方法
都对应一个artmethod结构体，每次要执行的时候，都会去这个结构体中，找到入口地址，然后跳转过去，但是这个结构体不能乱修改字段，会导致
非预期的结果，所有阿里的方式就是直接替换artmethod这个结构体，这样就可以直接控制程序流程，跳转到我们想要的入口了

# 2021年7月9日
1. 学了些安卓vmp的黑盒测试手法，也理解了些关于安卓vmp和armvmp的区别
，armvmp根据题目和实战中还是发现是通过花指令混淆加上打乱指令流实现，
安卓vmp，是java层的smail指令的解释执行，目前主要是两种方案，一种是数字的那种oncrate方法直接native化
，其实就是为了方便开发，毕竟签名一致，很容易把所有的oncrate方法都动态注册到一个函数中去解释执行，另一种
是所有方法都抽象成一个大函数，然后在大函数中进行动态注册加解释执行，那么黑盒的思路就是从invoke指令
出发，invoke在解释执行中对应功能就是执行方法功能，那么在jni中的体现就是findclass，
然后getmethodid，callxxx，这步的流程是逃不过的，所以我们可以hook那个方法，使用hook框架，或者
自己定制一个art的沙箱，我建议是沙箱，因为hook框架要考虑反调试的问题，而且不稳定，而且沙箱深度定制的话，
真的猛，牛逼，尤其对art非常了解，简直是降维打击
